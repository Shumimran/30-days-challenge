## 2. Vibe Coding vs Specification-Driven Development

* **Why does Vibe Coding usually create problems after one week?**

**Ans.**Vibe coding problems is liye banata hai kyunki shuru mein plan nahi hota, code messy ho jata hai, aur project barhne par sab cheezain manage karna mushkil ho jata hai.

* **How would Specification-Driven Development prevent those problems?**

**Ans.** SDD me pehle clear likh liya jata hai ke system ko kya karna hai.
Is se coding shuru honay se pehle hi requirements fix ho jati hain.
Result: less confusion, less rework, less bugs, aur team ko pata hota hai ke exact kya banana hai.

## 3. Architecture Thinking
* **How does architecture-first thinking change the role of a developer in AIDD?**

**Ans.** Architecture-first thinking developer ka role change kar deta hai coder se system designer banne mein.Is tarah developer pehle system ka clear design banata hai, phir coding karta hai.AIDD mein developer basically AI ko strong design ke saath guide karta hai.

* **Explain why developers must think in layers and systems instead of raw code.**

**Ans.** Jab aap sirf raw code mein sochte ho, to har cheez messy, slow aur confuse ho jati hai.
Layers aur systems aapko help karte hain ke aap project ko chhote, clear parts mein divide karo â€” jaise UI layer, logic layer, data layer.
Is se code samajhna, test karna, reuse karna aur bug fix karna 100x easy ho jata hai.
System thinking developer ko big picture dekhna sikhata hai, sirf lines of code nahi.
